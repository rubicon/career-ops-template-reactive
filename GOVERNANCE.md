# Governance

This project uses a lightweight benevolent-dictator model.

## Roles

- **Maintainer**: Dax Davis. Final decision maker on scope, design, and releases.
- **Contributors**: anyone who opens an issue or a pull request.

## How decisions are made

Most decisions happen in issues and pull requests in the open. The maintainer
reviews and merges contributions, sets direction, and cuts releases. When there
is disagreement, the maintainer makes the call and explains the reasoning.

The guiding principle is scope discipline: this is a career-ops template pack of
single-column, ATS-safe CV and cover-letter designs. Changes that keep the
templates parser-safe and the validator green are the easy yes.

## Changing this model

If the project grows enough to need more structure, this document will be updated
in a pull request before any change takes effect.
